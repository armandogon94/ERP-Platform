## Critical — fix before next /ship

### C-1. Cross-company FK writes bypass tenant isolation
`backend/modules/invoicing/serializers.py:47`, `backend/modules/sales/serializers.py:26,69`, `backend/modules/helpdesk/serializers.py:62`, `backend/modules/calendar/serializers.py:54-56`, `backend/modules/manufacturing/serializers.py` (bom), `backend/modules/projects/models.py:21`, `backend/modules/pos/models.py:60`

Writable `PrimaryKeyRelatedField`s default to a **global** queryset.
`CompanyScopedFilterBackend` only filters *reads*; writes are unvalidated.
A logged-in Company A user can `POST /invoices/ { "customer": <company_B_partner_id> }`
and the invoice will persist with a cross-tenant FK.

**Fix:** override `get_fields()` per serializer to constrain every FK queryset
to `<Model>.objects.filter(company=self.context["request"].company)`, or add
a `validate_<fk>()` per serializer. A tiny `TenantScopedPKRelatedField` mixin
can handle this in one place.

### C-2. Partner DELETE not admin-gated
`backend/core/views.py:135-155`

`PartnerViewSet` uses `IsCompanyMember` only. Any logged-in member can
`DELETE /partners/<id>/`, cascading into invoices, sales orders, tickets,
projects. Destructive ops must require admin.

**Fix:** override `get_permissions()` to require `IsCompanyAdmin` for
`destroy` (and probably `update`/`partial_update`).

### C-3. `pagination_class = None` on every ViewSet
`backend/modules/invoicing/viewsets.py:19`, `sales/viewsets.py:19,40,63`, `purchasing/viewsets.py:21,39,65,83,103`, `helpdesk/viewsets.py:28,39,62,120`, `core/views.py:142` (58 ViewSets across 14 files)

Global cursor pagination + `PAGE_SIZE=50` is configured
(`backend/config/settings/base.py:123-124`), but every module opts out.
With demo seeds this is invisible; at production scale, one request to
`/invoices/` returns the entire table.

**Fix:** drop the `pagination_class = None` overrides. Where a list must
stay un-paginated (e.g. AppSwitcher module list), apply pagination = None
explicitly with a comment.

### C-4. Invoice-posted notification fires on every save, not on status transition
`backend/core/signals.py:108-120`

Handler is `if instance.status != "posted": return`, so **every subsequent
save** of an already-posted invoice creates another notification for every
admin. Edits, re-computes, and seed idempotency runs all spam the bell.

**Fix:** compare against prior status via `pre_save` or
`Model.objects.only("status").get(pk=pk).status` inside `post_save`, and
notify only on transitions `!= "posted" → "posted"`.

### C-5. Open-sales-orders KPI always undercounts
`backend/core/views.py:240-242`

Filter is `status__in=["draft", "confirmed"]`, but `SalesOrder.Status`
starts at `CONFIRMED` and includes `IN_PROGRESS` (`modules/sales/models.py:55-60`).
`"draft"` never matches; `"in_progress"` orders aren't counted.

**Fix:** `status__in=["confirmed", "in_progress"]`.

### C-6. Synchronous AuditLog fan-out has no TTL or offload
`backend/core/signals.py:7-26`

Every `TenantModel.save()` → one `AuditLog.objects.create(...)` inside the
same request transaction. Slice 17 seed chains create ~300 rows per tenant
per seed run. No retention policy. Growth is unbounded.

**Fix:** buffer audit rows per-request and `bulk_create` on commit; add a
retention cron (90 days); consider offload to Celery.

### C-7. Ticket-create notification is O(N admins) sync inserts
`backend/core/signals.py:64-101`

Each Ticket create loops over `UserProfile.objects.filter(is_company_admin=True)`
and inserts one `Notification` per admin **inside the Ticket save
transaction**. 1 save = 1 + N inserts + N more AuditLog rows (cascade
through `create_audit_log`).

**Fix:** `Notification.objects.bulk_create([...])`; long-term move to a
Celery task.

### C-8. Reports module has no HTTP-layer tests
`backend/modules/reports/` — 482 LOC, 50 test LOC, **no `test_api.py`**.

The Slice 16 `/reports/pivot/`, `/reports/export/`, and scheduled-export
endpoints are entirely unexercised.

**Fix:** add `backend/modules/reports/tests/test_api.py` covering happy-path
+ whitelist rejection + company isolation.

### C-9. Ticket-update spam regression untested
`backend/core/tests/test_notifications.py`

The `if not created: return` guard at `core/signals.py:91` is the only thing
preventing every ticket edit from spamming admins. Zero regression test.

**Fix:** add `test_ticket_update_does_not_create_notification` that saves a
Ticket twice and asserts `Notification.count() == 1`.

### C-10. Calendar LWW tie-case untested
`backend/modules/calendar/viewsets.py:103`

Comparison is strict `incoming < stored`, so when timestamps are **equal**,
incoming wins (overwrites). Any timezone-rounding or clock-skew can flip
the outcome between runs. No tie test.

**Fix:** add `test_lww_equal_timestamp_prefers_stored_or_incoming` to pin
the documented direction.

## Important

### I-1. Bulk calendar upsert runs without a transaction
`backend/modules/calendar/viewsets.py:140-169` — partial failure leaves
some events persisted. Wrap loop in `transaction.atomic()` or use savepoints.

### I-2. LWW treats missing `updated_at` as "always win"
`backend/modules/calendar/viewsets.py:102-113` — `None` skips the compare,
stale payload silently overwrites newer stored data. Reject payloads
without `updated_at` on upsert, or treat missing as "stored wins".

### I-3. Module-config PATCH uses inline admin check
`backend/core/views.py:62-66` — raises `PermissionDenied` inside
`_patch_config` after body parsing. Move to `get_permissions()` returning
`[IsCompanyMember, IsCompanyAdmin]` when `action == "config"` and
`request.method == "PATCH"`.

### I-4. No `UniqueConstraint` on (company, external_uid) for Event
`backend/modules/calendar/models.py` — concurrent bulk POSTs can create
duplicates. Add
`UniqueConstraint(fields=["company","external_uid"], condition=Q(external_uid__gt=""))`.

### I-5. `updated_since` parser swallows malformed input
`backend/modules/calendar/viewsets.py:24-29` — `parse_datetime` returns
`None` silently, filter is skipped, caller gets full set. Return 400 when
present but unparseable.

### I-6. Blanket `except Exception: pass` in home_kpis
`backend/core/views.py:209-307` — hides real ORM/config bugs behind
zero-tiles (this is how C-5 went undetected). Catch only `ImportError`/
`LookupError` for missing modules; let query errors surface.

### I-7. AuditLog created-by internal saves pollutes timeline
`backend/core/signals.py:7-26` — backfill migrations and sequence-gen
internal `super().save()` calls produce audit rows with
`new_values={"saved": True}`. With the 100-row cap, real user actions can
be pushed off the visible tail. Gate on an explicit "user-initiated" flag
or `kwargs.get("raw")`.

### I-8. Partner FK list queries are missing `select_related`
`backend/modules/invoicing/viewsets.py:18`, `modules/sales/viewsets.py:18,39`,
`modules/purchasing/viewsets.py:38,82` — denormalized `customer_name`/
`vendor_name` makes the N+1 invisible today, but the moment anything
reads `.customer.<field>` the DB cost explodes. Add
`.select_related("customer")` / `("vendor", "partner")`.

### I-9. Frontend notifications polls twice every 30s
`frontend/src/hooks/useNotifications.ts:15-22` — `Promise.all([list, count])`
double-hits the API. List already returns `is_read`; compute count
client-side, or return `{results, unread_count}` envelope.

### I-10. `customer_name` CharField still coexists with Partner FK
`backend/modules/invoicing/models.py:40`, `modules/sales/models.py:27,70`
— the Slice 10.6 migration backfilled but never dropped the string column.
Dual-write paths remain in serializers. Finish the migration (drop the
CharField) or document the denormalization as intentional in DECISION.md.

### I-11. `InvoiceLine.total_price` is stored but not computed
`backend/modules/invoicing/models.py:62-83` — nothing enforces
`total = quantity * unit_price * (1 + tax_rate/100)`. A caller forgetting
to populate it silently corrupts totals. Compute in `save()` or the
serializer.

### I-12. `get_queryset` filter-params boilerplate repeated ~58 times
`modules/fleet/viewsets.py` (5×), `modules/projects/viewsets.py` (4×),
`modules/manufacturing/viewsets.py` (4×), `modules/helpdesk/viewsets.py`
(3×), etc. Extract a `FilterParamsMixin` with a declarative
`filter_params = {"status": "status", "driver": "driver_id"}`. ~200 LOC win.

### I-13. ReportListPage + reports module missing one test each
`frontend/src/pages/reports/ReportListPage.tsx` — no `.test.tsx` sibling.
`backend/modules/reports/` — no `test_api.py`. Both resolvable in one
commit (see C-8).

### I-14. Signal pattern inconsistency
Slice 10.7 brief said "signal auto-generation" but every module put it in a
`save()` override (`invoicing/models.py:53`, `sales/models.py:43,94`,
`purchasing/models.py:77,114`, `pos/models.py:77`, `helpdesk/models.py:123`,
`accounting/models.py:102`). `save()` is fine (simpler than `pre_save`),
but the slice doc should be updated so future modules know the canon.

## Suggestion

### S-1. Cache `home_kpis` per-company for 60 s
`backend/core/views.py:209-307` — `cache.get_or_set(f"home_kpis:{company.id}", ..., 60)`
cuts DB load on every dashboard refresh.

### S-2. `home_kpis` should be table-driven
6 near-identical try/except blocks (97 LOC) collapse to one `for` loop over
a `KPI_DEFS` list. Adding a new tile becomes a 1-line change.

### S-3. `_register_notification_signals` same reduction
`backend/core/signals.py:84-122` — registry list `[(model_path, filter_fn,
title_fn, type)]`. ~40 → 15 lines.

### S-4. Extract a `<ListPageShell>` React component
28 list pages repeat the `{loading && <Skeleton/>} {error && <Alert/>}
{!loading && !error && <table>...}` block. A shell composing the existing
`Skeleton` + `EmptyState` removes ~400 LOC of duplication.

### S-5. Lazy-load chart subtrees
`frontend/src/views/GraphView.tsx:1-17` imports Bar/Line/Pie/Area eagerly.
`React.lazy` keyed on `chartType` shrinks the initial bundle (~100 KB gz).

### S-6. Hard-coded hex colors leak past tokens
`globals.css:356-358`, `:414-415`, `:615,622` — bring audit-action chips,
notification-badge red, and error-boundary danger into `tokens.css`.
`GraphView.tsx:29-40` duplicates the 10 industry brand colors from
CLAUDE.md — pull from config.

### S-7. `GraphView` ternary chain → lookup map
`frontend/src/views/GraphView.tsx:52-108` — replace 4-branch nested ternary
with `{ bar: BarChart, line: LineChart, ... }` + one shared cartesian
block. ~30% shorter.

### S-8. Rate-limit `/calendar/events/bulk/`
500-item cap is fine, but add `UserRateThrottle` (~60/min).

### S-9. Seed commands → `bulk_create(ignore_conflicts=True)` once N > 50
Today's seeds are all < 10 rows per entity; safe for now but flag for when
per-industry counts grow.

### S-10. Projects page-triple gaps
`frontend/src/pages/projects/` has list pages for Task + Milestone but no
Form pages. Helpdesk has `CategoryListPage` with no form. Either
intentional (inline-only creation) or a gap — add a doc line.

### S-11. `core/models.py` at 544 LOC — split candidate
Mixes Company/User/Role/Partner/Notification/AuditLog/ModuleRegistry/
ViewDefinition. Not urgent (< 700 LOC) but worth splitting into
`core/models/{tenant,auth,partner,notification,registry}.py` before growth.

## Summary of what went well

- **Module Scaffold Pattern (D28)** produced 5 consecutive modules
  (Fleet → Projects → Manufacturing → POS → Helpdesk) with identical shape
  and **zero regressions**. Naming consistency across `{entity}_number`
  fields is clean.
- **AggregationMixin (D23)** uses PostgreSQL-native `values().annotate()` —
  no Python-side grouping fallbacks. Security whitelists are correctly
  enforced.
- **Design system (D34)** eliminated hard-coded hex in components
  (one small leak remaining — S-6).
- **Zero flaky test patterns** across 988 tests. No `time.sleep`, no raw
  `new Date()`, no unmocked `setTimeout`.
- **Preview sweeps** across DentaFlow (cyan) and NovaPay (blue) confirmed
  per-company theming, breadcrumbs, KPI tiles, notification bell all work
  end-to-end with zero console errors post-login.
- **No secrets committed.** SECRET_KEY uses env var with dev default;
  `admin/admin` only in dev seed.
